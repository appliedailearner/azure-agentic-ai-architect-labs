# v0.2.0-alpha Plan

## Target Outcome

`v0.2.0-alpha` should represent a move from alpha structure to stronger deployment realism and control wiring.

## Planned Focus Areas

- actual policy-assignment wiring in module 09
- credential-aware Azure deployment validation in CI
- stronger non-prod deployment verification
- additional module promotion only if it adds real depth

## Promotion Standard

`v0.2.0-alpha` should only be cut when:

- smoke validation is still passing
- release-readiness workflow is in place
- Azure deployment validation workflow is usable where credentials are configured
- module 09 deployment path is more realistic than `v0.1.0-alpha`
